# Feishu 普通用户使用指南

本文面向普通用户，目标是让你在飞书中完成完整评审闭环：发起评审 -> 查看结果 -> 回答澄清 -> 进入下一步动作。  
如果你是管理员，请先阅读 [feishu-setup.md](./feishu-setup.md)。

## 你可以在飞书里完成什么

- 提交一份 PRD/需求文档进行评审
- 在飞书内直接打开评审结果
- 按提示回答澄清问题并刷新结果
- 从结果页继续进入下一步交付动作

## 普通用户操作流程

### 第一步：进入飞书评审入口

进入团队提供的飞书评审入口页面，通常由管理员配置为应用内链接。  
如果你不确定入口位置，联系团队管理员获取入口地址。

### 第二步：提交待评审内容（发起 PRD 评审）

在提交页中填写或选择待评审内容后，点击提交。  
提交成功后，系统会开始生成评审结果。

### 第三步：查看飞书内结果页

在飞书内打开结果页（通常为 `.../run/<run_id>?embed=feishu...`），你会看到：

- 评审结论摘要
- 关键问题与风险
- 后续建议

如果结果还在生成中，页面会自动展示当前进度。

### 第四步：回答澄清问题（如有）

当页面提示“需要补充信息”时，按问题逐项回答并提交。  
系统会基于补充信息更新结果，你可以继续在同一页面查看最新版本。

### 第五步：继续下一步动作

当结果稳定后，可直接在结果页执行下一步动作，例如：

- 进入下一步交付区
- 下载产物给相关角色
- 再次发起评审迭代

### 第六步：确认最终结果并分享

确认结果满足预期后，可将页面或导出报告分享给相关同事继续推进。

## 常见问题

### 1) 提交后一直显示处理中怎么办？

- 先刷新页面并等待一段时间
- 确认文档内容可访问且未被删除
- 如仍无结果，联系管理员排查服务状态

### 2) 为什么我看不到结果页？

- 可能是入口链接已过期或缺少权限
- 尝试从飞书应用入口重新进入
- 若仍失败，请联系管理员确认访问配置

### 3) 回答澄清后结果没有变化怎么办？

- 检查是否已真正提交成功
- 适当补充更具体的信息后再提交一次
- 仍无变化时，联系管理员协助排查

### 4) 哪些文档更适合提交评审？

优先提交目标明确、范围清晰、包含关键约束的 PRD/需求文档。  
信息越完整，评审结论越稳定。
