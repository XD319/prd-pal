# Feishu 演示材料说明

本文定义飞书主入口演示素材的最小集合、命名规范和拍摄脚本，便于产品、运营、实施团队复用同一套演示资产。

## 1. 素材目标

演示需要覆盖完整用户闭环：

1. 在飞书发起评审
2. 在飞书内查看结果页
3. 回答澄清问题
4. 继续进入下一步动作

## 2. 建议素材清单

建议至少准备以下 6 份文件（`docs/assets/feishu/`）：

1. `feishu-entry-open.gif`：从飞书应用入口打开 `Feishu 工作入口`
2. `feishu-submit-review.gif`：填写 PRD 来源并提交评审
3. `feishu-result-overview.png`：结果页关键区域总览
4. `feishu-clarification-flow.gif`：回答澄清并刷新结果
5. `feishu-next-delivery.gif`：从结果页进入下一步交付动作
6. `feishu-common-error.png`：常见错误提示（权限、上下文缺失）

## 3. 命名与版本

- 使用小写中划线命名：`feishu-<scene>-<action>.<ext>`
- 文件名不带日期；版本通过目录管理：
  - `docs/assets/feishu/v1/`
  - `docs/assets/feishu/v2/`
- 每次更新后在同目录补 `README.md`，记录：
  - 录制环境（域名、版本、语言）
  - 对应代码分支
  - 关键脚本差异

## 4. 拍摄脚本（推荐）

### 脚本 A：首次提交到结果可见（30-45 秒）

1. 打开飞书入口 `.../feishu`
2. 粘贴飞书文档链接并提交
3. 展示 run 创建成功与“查看最新结果”
4. 停留在结果摘要、风险、关键发现

### 脚本 B：澄清回填（20-30 秒）

1. 在结果页定位到澄清区
2. 提交一条澄清回答
3. 展示状态更新与结果刷新

### 脚本 C：下一步交付（15-25 秒）

1. 在结果页点击“生成下一步交付”
2. 展示交付区或产物下载区域
3. 说明可继续 handoff 给执行角色

### 脚本 D：异常提示（10-20 秒）

1. 演示权限不足或上下文缺失
2. 展示用户可理解的错误文案
3. 指出“联系管理员”的处理路径

## 5. 发布前检查

- 画面中不出现真实敏感信息（租户、用户、密钥）
- 文案与当前 UI 一致（按钮名称、标题）
- 结果页链接示例使用 `embed=feishu`
- 移动端截图至少一张（飞书内 H5 可读性）
