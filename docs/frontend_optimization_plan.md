## 前端优化执行方案（Requirement Review Frontend）

本文档基于当前前端实现现状整理可落地的优化项，目标是先解决稳定性、可维护性和用户感知问题，再考虑更重的工程化升级。

---

## 一、评估结论

当前前端的核心问题不是视觉层，而是 `App` 同时承担了过多运行逻辑，导致：

- 运行态切换、历史任务回看、WebSocket 与轮询兜底逻辑集中在一个组件内。
- 轮询、WebSocket、fallback timeout、`localStorage` 等副作用分散管理，清理边界不够明确。
- 用户错误反馈主要依赖日志，表单和主界面缺少显式提示。
- 代码已具备继续组件化演进的基础，但仍缺少 hook、API、常量层的隔离。

这意味着后续优化应优先围绕“稳定运行”和“降低耦合”，而不是直接推进完整状态机或 TypeScript 全量迁移。

---

## 二、确认保留的优化方向

以下建议与当前代码结构匹配，应该纳入本轮优化：

### 1. 收敛运行时副作用

- 将任务创建、状态轮询、WebSocket 连接、报告加载、清理逻辑集中到 `useReviewRun`。
- 将历史持久化和日志维护从页面组件中拆出，避免 `App` 继续膨胀。

### 2. 抽离网络与常量层

- 抽出 `api.js`，统一管理 `/api/review`、`/api/report` 等接口访问。
- 抽出 `constants.js`，集中管理轮询间隔、fallback 超时、节点列表、终态集合、历史记录上限等常量。

### 3. 拆分主要 UI 面板

- 将提交区、运行状态区、报告区、历史区拆成独立组件。
- `App` 保留为页面装配层，只负责组合状态和组件。

### 4. 增强错误反馈与可访问性

- 表单空值、请求失败、报告加载失败等场景需要显式错误提示。
- 为输入模式切换补齐 ARIA 属性，提升键盘操作和屏幕阅读器可用性。

### 5. 补充轻量性能保护

- 限制日志最大条数，避免长时运行导致列表无限增长。
- 对历史记录读取增加结构校验，避免异常 `localStorage` 污染页面状态。

---

## 三、对原方案的修正

以下内容保留为后续可选项，但不作为本轮主目标：

### 1. `useReducer` / 完整状态机

- 这类方案在复杂度更高时很有价值，但当前页面规模仍适合先通过 hook 拆分和少量结构化状态来降复杂度。
- 本轮只需要为运行态建立更清晰的清理和切换边界，不必立即引入完整 reducer 状态机。

### 2. TypeScript 迁移

- 类型化建模对长期维护有益，但当前主要痛点不是类型缺失，而是逻辑耦合和副作用分散。
- 本轮先完成模块分层和稳定性修复，后续再视需要迁移到 TypeScript。

### 3. 历史筛选与日志折叠

- 属于增强体验，不是当前关键矛盾。
- 在核心运行链路稳定之前，不进入首批实施范围。

---

## 四、本轮最终优化清单

### 第一优先级：运行态稳定性

- 统一管理 polling、WebSocket、fallback timeout 的创建与清理。
- 在重新提交任务或打开历史任务前，先停止旧监听。
- 为 WebSocket 消息解析、网络请求失败等异常路径增加保护。

### 第一优先级：用户可感知错误反馈

- 为 PRD 文本和文件路径输入增加表单级错误提示。
- 为任务创建失败、轮询失败、报告预览失败增加显式错误区域。
- 根据运行状态调整按钮可用性和提示文案。

### 第二优先级：副作用与数据访问解耦

- 提取 `useReviewRun`、`useLogs`、`useHistory`。
- 提取 `api.js` 与 `constants.js`。
- 将运行逻辑从页面 JSX 中抽离，降低 `App` 复杂度。

### 第二优先级：组件拆分

- 拆分 `InputPanel`、`RunStatusPanel`、`ReportPanel`、`HistoryPanel`。
- 保持面板组件偏展示层，运行逻辑继续留在 hook。

### 第三优先级：可访问性与轻量性能

- 为 mode 切换增加 `role="tab"`、`aria-selected`、`aria-controls` 等属性。
- 限制日志保留条数，例如最多 200 条。
- 对历史记录读写增加结构校验与裁剪。

---

## 五、实施顺序

综合改动风险与收益，按以下顺序落地：

1. 修复运行态稳定性问题。
2. 增加表单与请求错误的显式 UI 反馈。
3. 提取 hooks、API、常量模块。
4. 拆分主要 UI 组件。
5. 补齐可访问性和轻量性能保护。

---

## 六、暂缓项

以下内容不进入本轮实施：

- 完整的 reducer 状态机重构。
- TypeScript 全量迁移。
- 历史筛选、日志折叠等增强交互。
