# PRD — 商户运营后台 v1.0

## 背景

公司当前通过 Excel + 企业微信手动管理线下商户的入驻、合同续签和结算对账流程。随着商户规模从 200 家增长至 1,500 家，人工操作频繁出错且无法追溯。需要建设一套 Web 端商户运营后台，覆盖商户全生命周期管理，降低运营人力成本并提升数据准确性。

## 功能列表

### F-1 商户入驻审批

- 运营人员创建商户入驻申请，上传营业执照、法人身份证等资质材料。
- 审批流程支持两级审核：运营主管 → 风控专员。
- 审批通过后系统自动生成商户编号，并发送入驻成功短信通知。

### F-2 合同管理

- 支持在线生成标准合同模板，自动填充商户名称、合作期限、费率等字段。
- 合同到期前 30 天自动触发续签提醒（站内消息 + 邮件）。
- 提供合同变更记录查询，支持按商户名称和合同编号检索。

### F-3 结算对账

- 每月 T+1 日自动从交易系统拉取上月交易流水，生成结算单。
- 运营人员可在线核对结算金额，标记异常订单。
- 结算单经财务确认后进入付款队列，付款状态实时同步。

### F-4 数据看板

- 展示商户总数、本月新增/流失数、平均结算周期等核心指标。
- 支持按地区、商户类型、合作时长进行多维筛选。

## 非功能需求

- **性能**：商户列表页在 5,000 条数据量下，首屏加载 ≤ 2 秒。
- **安全**：所有管理操作须通过 RBAC 权限控制，敏感字段（身份证号、银行账号）脱敏展示。
- **审计**：关键操作（审批、合同变更、结算确认）记录完整操作日志，保留 3 年。
- **可用性**：系统可用性 ≥ 99.5%（月度），计划内维护窗口需提前 48 小时通知。

## 约束与开放问题

| # | 类型 | 描述 |
|---|------|------|
| 1 | 约束 | 必须对接现有交易系统 REST API（v2.3），接口文档已提供。 |
| 2 | 约束 | 第一期仅支持 PC Web 端，移动端在 v2.0 规划。 |
| 3 | 开放问题 | 合同电子签章供应商尚未确定（候选：e签宝 / 法大大），需产品侧决策后确认集成方案。 |
| 4 | 开放问题 | 结算异常的自动检测规则是否由风控团队提供，还是由运营人员手动配置？ |
